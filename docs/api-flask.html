Estrutura mínima do projeto

Antes de pensar nas rotas, a API precisa deixar claro onde cada responsabilidade mora: inicialização, conexão, models, controllers e arquivos de rota.

app.py

Ponto de entrada da API. Cria a app, carrega config e registra os blueprints.

app/models/

Representa as tabelas e relacionamentos. Aqui ficam `user.py` e `task.py`.

app/controllers/

Recebe a regra de negócio. Cada endpoint chama uma função de controller.

app/routes/

Mantém os blueprints e o mapeamento entre URL e controller.

Flask

Cria a aplicação, recebe requests, devolve responses e registra as rotas.

Flask-SQLAlchemy

Mapeia classes em tabelas e facilita consultas, filtros e agregações.

Flask-JWT-Extended

Gera e valida tokens JWT para proteger as rotas privadas.

Werkzeug

Entra no hash de senha com `generate_password_hash` e `check_password_hash`.

python-dotenv

Carrega variáveis do `.env` para tirar segredos do código.

datetime + func

Convertem datas e calculam totais diretamente no banco.

Config e conexão

O `.env` continua guardando segredos, mas a conexão não deve ficar espalhada. Centralize configuração em `config.py` e a inicialização compartilhada em `app/extensions.py`.

Guarde no `.env` a conexão com o banco, `JWT_SECRET_KEY`, `SECRET_KEY` e flags de ambiente. O arquivo real não deve ir para o git.

Extensões compartilhadas

from flask_jwt_extended import JWTManager
from flask_sqlalchemy import SQLAlchemy

db = SQLAlchemy()
jwt = JWTManager()

def init_extensions(app):
    db.init_app(app)
    jwt.init_app(app)

Cadastro e Login

O fluxo de autenticação fica mais limpo quando o model do usuário, o controller e a rota estão em arquivos separados.

app/models/user.py

from app.extensions import db

class User(db.Model):
    __tablename__ = "users"

    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True, nullable=False)
    password_hash = db.Column(db.String(255), nullable=False)

app/controllers/auth_controller.py

from flask import request
from flask_jwt_extended import create_access_token
from werkzeug.security import check_password_hash, generate_password_hash

from app.extensions import db
from app.models.user import User

def register():
    data = request.get_json()
    user = User(
        username=data["username"],
        password_hash=generate_password_hash(data["password"])
    )
    db.session.add(user)
    db.session.commit()
    return {"message": "usuário criado"}, 201

def login():
    data = request.get_json()
    user = User.query.filter_by(username=data["username"]).first()

    if not user or not check_password_hash(user.password_hash, data["password"]):
        return {"message": "credenciais inválidas"}, 401

    token = create_access_token(identity=str(user.id))
    return {"token": token}, 200

app/routes/auth_routes.py

from flask import Blueprint

from app.controllers.auth_controller import login, register

auth_bp = Blueprint("auth", __name__, url_prefix="/auth")

auth_bp.post("/register")(register)
auth_bp.post("/login")(login)

Criar tarefas

Repita o mesmo padrão nas tarefas: model para a tabela, controller para a regra e um arquivo de rota para expor o endpoint.

app/models/task.py

from app.extensions import db

class Task(db.Model):
    __tablename__ = "tasks"

    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(120), nullable=False)
    date = db.Column(db.Date, nullable=False)
    value = db.Column(db.Float, default=0)
    user_id = db.Column(db.Integer, db.ForeignKey("users.id"), nullable=False)

app/controllers/task_controller.py

from datetime import datetime

from flask import request
from flask_jwt_extended import get_jwt_identity, jwt_required

from app.extensions import db
from app.models.task import Task

@jwt_required()
def create_task():
    data = request.get_json()
    task = Task(
        title=data["title"],
        date=datetime.strptime(data["date"], "%Y-%m-%d").date(),
        value=float(data.get("value", 0)),
        user_id=int(get_jwt_identity())
    )
    db.session.add(task)
    db.session.commit()
    return {"message": "tarefa registrada"}, 201

app/routes/task_routes.py

from flask import Blueprint

from app.controllers.task_controller import create_task, get_tasks_summary

task_bp = Blueprint("tasks", __name__, url_prefix="/tasks")

task_bp.post("/")(create_task)
task_bp.get("/summary")(get_tasks_summary)

Cálculo por usuário e data

Esse item foi adicionado para deixar explícito o cálculo: primeiro filtra por usuário, depois agrupa por data, e só então soma e conta os registros.

1. Usuário

`get_jwt_identity()` devolve o ID do dono do token e protege o resumo.

2. Data

`group_by(Task.date)` junta os registros do mesmo dia.

3. Total

`count()` conta tarefas e `sum()` devolve o valor consolidado.

app/controllers/task_controller.py

from sqlalchemy import func

@jwt_required()
def get_tasks_summary():
    current_user_id = get_jwt_identity()
    rows = (
        db.session.query(
            Task.date,
            func.count(Task.id).label("tasks_count"),
            func.sum(Task.value).label("total_value")
        )
        .filter(Task.user_id == current_user_id)
        .group_by(Task.date)
        .all()
    )
    return {
        "daily_summary": [
            {
                "date": row.date.strftime("%Y-%m-%d"),
                "tasks_count": row.tasks_count,
                "total_value": row.total_value or 0
            }
            for row in rows
        ]
    }, 200
