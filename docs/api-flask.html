Rotas e Blueprints no Flask

Rota é o endereço HTTP que chama uma função. Blueprint é o conjunto de rotas relacionadas, registrado com um prefixo para manter o projeto limpo.

Rota

Exemplo: `POST /auth/login`. Ela recebe dados, executa uma regra e devolve uma resposta JSON.

Blueprint

Exemplo: `auth_bp = Blueprint("auth", __name__, url_prefix="/auth")`. Todas as rotas de login e cadastro ficam juntas.

project/
  app.py
  routes/
    auth.py
    tasks.py

auth_bp = Blueprint("auth", __name__, url_prefix="/auth")
tasks_bp = Blueprint("tasks", __name__, url_prefix="/tasks")

app.register_blueprint(auth_bp)
app.register_blueprint(tasks_bp)

.env e a importância dele

Esse menu foi criado para explicar por que configuração sensível não deve ficar hardcoded em `app.py`.

Guarde no `.env` a conexão com o banco, `JWT_SECRET_KEY`, `SECRET_KEY` e flags de ambiente. O arquivo real não deve ir para o git.

Exemplo de `.env`

FLASK_ENV=development
DATABASE_URL=sqlite:///banco.db
JWT_SECRET_KEY=troque-esta-chave

Leitura no app

from dotenv import load_dotenv
import os

load_dotenv()
app.config["SQLALCHEMY_DATABASE_URI"] = os.getenv("DATABASE_URL")
app.config["JWT_SECRET_KEY"] = os.getenv("JWT_SECRET_KEY")

Cadastro e Login

Esse menu explica como o usuário entra no sistema e como a API passa a reconhecer a identidade dele nas rotas protegidas.

Cadastro

@auth_bp.post("/register")
def register():
    data = request.get_json()
    user = User(username=data["username"])
    user.password_hash = generate_password_hash(data["password"])
    db.session.add(user)
    db.session.commit()
    return {"message": "usuário criado"}, 201

Login

@auth_bp.post("/login")
def login():
    data = request.get_json()
    user = User.query.filter_by(username=data["username"]).first()
    token = create_access_token(identity=str(user.id))
    return {"token": token}, 200

Criar tarefas

Esse menu detalha como gravar título, descrição, data e valor sempre vinculando a tarefa ao usuário autenticado.

@tasks_bp.post("/")
@jwt_required()
def create_task():
    current_user_id = get_jwt_identity()
    data = request.get_json()
    task = Task(
        title=data["title"],
        date=datetime.strptime(data["date"], "%Y-%m-%d").date(),
        value=float(data.get("value", 0)),
        user_id=current_user_id
    )
    db.session.add(task)
    db.session.commit()
    return {"message": "tarefa registrada"}, 201

Cálculo por usuário e data

Esse item foi adicionado para deixar explícito o cálculo: primeiro filtra por usuário, depois agrupa por data, e só então soma e conta os registros.

1. Usuário

`get_jwt_identity()` devolve o ID do dono do token e protege o resumo.

2. Data

`group_by(Task.date)` junta os registros do mesmo dia.

3. Total

`count()` conta tarefas e `sum()` devolve o valor consolidado.

@tasks_bp.get("/summary")
@jwt_required()
def get_tasks_summary():
    current_user_id = get_jwt_identity()
    rows = (
        db.session.query(
            Task.date,
            func.count(Task.id).label("tasks_count"),
            func.sum(Task.value).label("total_value")
        )
        .filter(Task.user_id == current_user_id)
        .group_by(Task.date)
        .all()
    )
    return {
        "daily_summary": [
            {
                "date": row.date.strftime("%Y-%m-%d"),
                "tasks_count": row.tasks_count,
                "total_value": row.total_value or 0
            }
            for row in rows
        ]
    }, 200
