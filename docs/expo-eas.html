Introducao ao EAS

EAS padroniza build, credenciais e caminho ate a distribuicao.

Ele resolve a parte mais sensivel da esteira mobile: build consistente em nuvem, gerenciamento de assinatura, perfis de ambiente e integracao com distribuicao.

O que ele resolve

Build em nuvem: voce nao depende do mesmo setup local para cada pessoa do time.
Credenciais: o servico pode cuidar da assinatura Android e iOS.
Perfis: o eas.json separa desenvolvimento, preview e producao.
Distribuicao: voce sai mais rapido do codigo para testes internos e lojas.

Configuracao guiada

O caminho mais seguro e instalar, autenticar e so depois buildar.

A sequencia abaixo foi montada para virar um roteiro repetivel sempre que voce iniciar um novo projeto Expo.

1. Verifique o ambiente

Confirme Node.js e npm

node -v
npm -v

2. Crie a base do app

Suba um novo projeto Expo

npx create-expo-app@latest meu-app
cd meu-app

3. Instale a EAS CLI

Conecte o terminal ao servico

npm install -g eas-cli
eas login
eas whoami

4. Configure e build

Gere o eas.json e faca o primeiro preview

eas build:configure
npx expo install expo-dev-client
eas build --platform android --profile preview

Arquivo central

Exemplo minimo de eas.json

{
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal"
    },
    "production": {
      "autoIncrement": true
    }
  }
}
